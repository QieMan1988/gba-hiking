## 1. 核心玩法设计

### 1.1 游戏核心概念

一款以**大湾区地理文化**为背景，融合**登山式消除**、**轻Roguelike卡牌构筑**与**环保教育**的单人策略休闲游戏。

玩家通过"穿越"卡牌模拟徒步登山，在爽快操作中深度体验岭南自然人文之美。

**核心设计原则**：
- **玩法优先**：强调卡牌穿越的爽快感和策略深度
- **规则清晰**：所有机制边界明确，异常处理完善
- **体验量化**：爽快感、策略性、沉浸感、成长感四维可衡量
- **文化融入**：大湾区文化元素自然融入核心玩法，避免生硬植入

### 1.2 登山式卡牌布局

#### 1.2.1 布局结构

采用**正三角登山式布局**，从底部山脚到顶部山顶，层数递减。

**层级与徒步距离对应**：
- 每层代表约2公里的徒步距离
- 总徒步距离：10-26公里（根据关卡类型）
- 累积爬升：200-1200米（根据关卡难度）

**层数规则**：

| 关卡类型 | 总层数 | 总徒步距离 | 山顶奖励卡 | 每层卡牌数（从下到上） |
|---------|-------|-----------|-----------|---------------------|
| **新手关 (1-3)** | 5层 | 10公里 | 1张 | 4-3-3-2-1 |
| **澳门新手关 (1.5, 1.8)** | 3-4层 | 4.3-8.1公里 | 1张 | 4-3-1 或 4-3-3-1 |
| **标准关 (4-6)** | 7层 | 14公里 | 1张 | 4-4-4-4-3-2-1 |
| **挑战关 (7-9)** | 9层 | 18公里 | 1张 | 5-5-5-5-4-3-2-2-1 |
| **精英关 (10)** | 11层 | 22公里 | 1张 | 6-6-6-5-4-3-3-2-2-1-1 |
| **BOSS关 (11+)** | 13层 | 26公里 | 1张 | 特殊设计 |

#### 1.2.2 卡牌生成逻辑

**基础生成规则**：
1. **风景卡最低保障**：每层至少1张风景卡，保证连击可玩性
2. **卡牌排列规则**：禁止连续3张同类型卡牌，避免卡手
3. **地形障碍率**：根据游玩次数递增，范围15%-25%
4. **资源卡与环境卡**：根据关卡难度动态调整比例

**卡牌生成权重算法**：

```
P(card_type) = W_base × W_playcount × W_weather × W_faction

其中：
- W_base：基础权重（风景卡40%，地形卡30%，资源卡20%，环境卡10%）
- W_playcount：游玩次数权重（游玩次数越多，地形卡权重越高）
- W_weather：天气权重（高温时环境卡权重+50%，台风时地形卡权重+30%）
- W_faction：流派权重（玩家收集照片卡后，对应类型权重+20%）
```

#### 1.2.3 视觉呈现

每层都有明确的层级标注、徒步距离和累积爬升数值，增强"登山"的沉浸感。

```
第7层（山顶）：[山顶奖励卡 ★]  海拔：400-430m

第6层：2张卡  徒步距离：12-14km  累积爬升：400-430m

第5层：3张卡  徒步距离：10-12km  累积爬升：340-370m

第4层：4张卡  徒步距离：8-10km  累积爬升：310-340m  ← 当前操作层（边缘高亮）

第3层：4张卡  徒步距离：6-8km  累积爬升：280-310m  ← 已穿越层（半透明）

第2层：4张卡  徒步距离：4-6km  累积爬升：250-280m  ← 已穿越层（半透明）

第1层（山脚）：4张卡  徒步距离：2-4km  累积爬升：220-250m  ← 已穿越层（半透明）
```

**文化元素融入**：
- **累积爬升数值**：根据地形类型动态计算（如麦理浩径第7段650米爬升）
- **背景图**：每层背景切换为大湾区不同风景（维多利亚港、珠江口、岭南建筑等）
- **卡牌样式**：风景卡展示大湾区地标（大三巴牌坊、广州塔、港珠澳大桥等）

### 1.7 规则边界与异常处理

#### 1.7.1 属性系统边界（更新）

**体能边界**：

| 边界类型 | 边界值 | 处理规则 | 异常情况处理 |
|---------|-------|---------|------------|
| **最大值** | 100 | 超过100的部分不计入 | 无 |
| **最小值** | 0 | 达到0时游戏结束 | 体能降为负数：强制结束游戏 |
| **恢复上限** | 100 | 超过100的部分不计入 | 单次恢复>100：限制为100 |

**体能消耗公式**：
```
体能消耗 = (徒步距离 / 20) + (累积爬升 / 20)

其中：
- 徒步距离单位：公里
- 累积爬升单位：米
- 体能消耗结果：点
```

**基准设定**：
- 初始体能 100 = 20 公里徒步 或 2000 米累积爬升

## 3. 数值与平衡

### 3.1 五维属性系统

#### 3.1.1 属性定义

| 属性 | 取值范围 | 起始值 | 主要影响 |
|------|---------|--------|---------|
| **体能**<br>(主属性) | 0-100 | 100 | 视为生命值，为0则游戏结束。影响负重上限（公式：基础负重+体能*系数）。所有其他属性变化都会间接影响体能。 |
| **疲劳** | 0-20 | 0 | 体能降低的缓冲区。累积公式：`负重值 × 0.1 + 操作次数 × 0.05 + 心率影响`。高疲劳会降低所有行动效率。 |
| **饥饿** | 0-100 | 100 | 正向影响体能恢复速率。低于阈值时，体能恢复大幅减慢。 |
| **口渴** | 0-100 | 100 | 正向影响体能恢复速率。与饥饿共同作用，缺水影响更显著。 |
| **心率** | 80-180 | 110 | 反映运动强度。80-100无影响；100-180区间，心率越高，体能恢复速率越低，疲劳累积越快。 |

### 3.2 经济系统

#### 3.2.1 三种货币

| 货币类型 | 功能 | 局内/局外 | 获取方式 | 用途 |
|---------|------|----------|---------|------|
| **徒步数** | 局外经验 | 局外 | 徒步距离累积（每1公里=2,000徒步数） | 局外角色属性升级 |
| **累积爬升** | 局外稀有货币 | 局外 | 通关奖励（每关根据实际爬升计算） | 局外兑换高级装备、解锁装备槽位 |
| **环保值** | 局内货币 | 局内 | 消除环境卡、完成环保行为 | 局内商店购买物品+照片卡包 |

**徒步数计算公式**：
```
徒步数 = 徒步距离（公里）× 2,000

示例：
- 徒步10公里 = 10 × 2,000 = 20,000徒步数
- 徒步14公里 = 14 × 2,000 = 28,000徒步数
- 徒步26公里 = 26 × 2,000 = 52,000徒步数
```

**累积爬升计算**：
- 累积爬升 = Σ(所有上坡路段的垂直上升高度)
- 下坡路段的垂直下降高度不计入累积爬升中
- 平路（坡度<5%）不计入累积爬升

### 3.4 补给系统

#### 3.4.1 补给需求标准

**基准补给需求**：
- 每 10 公里徒步需要 2 瓶水 + 1 瓶运动饮料 + 1 次巧克力补给

**补给需求公式**：
```
需水量 = 徒步距离 / 5
需运动饮料量 = 徒步距离 / 10
需巧克力补给次数 = 徒步距离 / 10
```

#### 3.4.2 资源卡设计

| 资源卡名称 | 恢复效果 | 环保值消耗 | 补给单位 |
|-----------|---------|-----------|---------|
| **矿泉水** | 口渴+20 | 25环保值 | 1瓶 |
| **运动饮料** | 口渴+15，疲劳-5 | 50环保值 | 1瓶 |
| **巧克力** | 体能+15，饥饿+20 | 75环保值 | 1次 |
| **能量棒** | 体能+20，饥饿+15 | 100环保值 | 1次 |
| **香蕉** | 体能+10，钾元素+20 | 60环保值 | 1次 |
| **电解质水** | 口渴+30，心率-10 | 80环保值 | 1瓶 |
| **运动补给包** | 口渴+25，体能+20，疲劳-10 | 150环保值 | 1次（综合补给） |

#### 3.4.3 补给点设计

| 补给点类型 | 包含内容 | 价格（环保值） | 出现位置 |
|-----------|---------|--------------|---------|
| **小型补给点** | 1瓶矿泉水 + 1根能量棒 | 125环保值 | 新手关每2层 |
| **中型补给点** | 1瓶运动饮料 + 1次巧克力补给 | 125环保值 | 标准关每2-3层 |
| **大型补给点** | 1次运动补给包 + 1瓶矿泉水 | 175环保值 | 挑战关每3层 |
| **超级补给点** | 2次运动补给包 + 照片卡包（基础） | 300环保值 | 精英/Boss关每3层 |

#### 3.4.4 补给不足处理

**补给不足警告**：

```
当徒步距离达到5公里但补给不足时，显示警告：

⚠️ 补给不足警告
─────────────────────
你已经徒步了5公里，但补给不足！

当前补给状态：
- 水：0.8/1.0瓶 ❌
- 运动饮料：0.4/0.5瓶 ❌
- 巧克力：0.4/0.5次 ❌

后果：
- 口渴/饥饿恢复速率降低至5%
- 疲劳积累速度加快50%
- 体能消耗增加10%

建议：
- 在下一层寻找游动商贩或景区商店
- 合理使用环保值购买补给
```

### 3.5 疲劳系统

#### 3.5.1 疲劳机制

**基准疲劳设定**：
- 每 5 公里徒步需要 1 次休息缓解疲劳才能继续
- 游戏中后期可突破该限定（通过装备或照片卡效果）

**疲劳计算公式**：
```
疲劳积累 = (徒步距离 / 5) + (累积爬升 / 250)

其中：
- 徒步距离单位：公里
- 累积爬升单位：米
- 疲劳积累结果：次（需要休息的次数）
```

#### 3.5.2 休息机制

**休息次数限制**：

| 游戏阶段 | 初始休息次数 | 每次休息消耗 | 最大休息次数 |
|---------|-----------|------------|------------|
| **新手期（1-3关）** | 3次 | 1次 | 3次 |
| **标准期（4-6关）** | 2次 | 1次 | 2次 |
| **挑战期（7-11关）** | 1次 | 1次 | 1次 |
| **突破限制** | 0次 | - | 通过装备/照片卡实现 |

**休息效果**：

| 休息类型 | 恢复效果 | 消耗 | 备注 |
|---------|---------|------|------|
| **快速休息** | 疲劳-5 | 30秒 | 最小恢复 |
| **标准休息** | 疲劳-10，体能+20 | 60秒 | 标准恢复 |
| **深度休息** | 疲劳-15，体能+30，饥饿+20，口渴+20 | 90秒 | 最大恢复 |

#### 3.5.3 疲劳突破机制

### 4.2 关卡长度与距离设计

#### 4.2.1 关卡生成规则

**关卡生成流程**：

1. **确定关卡类型**：根据游玩次数和玩家经验
2. **确定总层数**：3-13层（根据关卡类型）
3. **生成地形序列**：确保上坡下坡的合理分布
4. **生成卡牌分布**：根据地形类型确定卡牌类型权重
5. **生成补给点**：根据距离和难度确定补给点位置

#### 4.2.2 地形生成规则

```
关卡地形生成算法：

1. 计算目标累积爬升：
   - 新手关：200-400m
   - 标准关：400-600m
   - 挑战关：600-800m
   - 精英关：800-1000m
   - BOSS关：1000-1200m

2. 生成地形序列：
   - 每层2公里
   - 确保上坡和下坡的合理交替
   - 避免连续3层都是陡坡

3. 示例（新手关5层）：
   - 第1层：平坦道路（0m爬升）
   - 第2层：缓坡上坡（50m爬升）
   - 第3层：平坦道路（0m爬升）
   - 第4层：缓坡上坡（50m爬升）
   - 第5层（山顶）：缓坡上坡（50m爬升）
   - 总累积爬升：150m
```

### 4.3 上坡下坡机制

#### 4.3.1 地形坡度识别系统

**关卡生成时，每层会被分配一个地形类型**：

| 地形类型 | 坡度 | 累积爬升（每层） | 下坡概率 | 特殊机制 |
|---------|------|----------------|---------|---------|
| **平坦道路** | 0-5% | 0m | 0% | 无特殊机制 |
| **缓坡上坡** | 5-15% | 100m | 10% | 体能消耗增加 |
| **陡坡上坡** | 15-30% | 300m | 15% | 疲劳增加，可能触发"呼吸急促" |
| **悬崖栈道** | >30% | 600m | 20% | 恐惧状态，需要确认翻越 |
| **缓坡下坡** | -15%到-5% | 0m（不计入） | 100% | 体能消耗减少，但可能触发"膝盖疼痛" |
| **陡坡下坡** | -30%到-15% | 0m（不计入） | 100% | 体能消耗大幅减少，但可能触发"膝盖磨损" |

#### 4.3.2 上坡机制

**上坡特征**：
- 计入累积爬升
- 体能消耗增加
- 疲劳积累加快
- 心率上升

**上坡视觉效果**：
- 层级显示：向上箭头🔼（红色表示陡坡，橙色表示缓坡）
- 背景渐变：从下方向上方颜色加深
- 海拔数值：显示当前累积爬升

**上坡地形障碍卡**：

| 卡牌名称 | 累积爬升（每张） | 体能消耗 | 特殊效果 |
|---------|----------------|---------|---------|
| **缓坡路段** | 50m | 8-10点 | 心率+5 |
| **陡坡路段** | 150m | 15-25点 | 疲劳+5，心率+10 |
| **悬崖栈道** | 300m | 20-30点 | 恐惧状态3秒，心率+15 |

#### 4.3.3 下坡机制

**下坡特征**：
- 不计入累积爬升
- 体能消耗减少
- 可能触发"膝盖疼痛"或"膝盖磨损"
- 速度加快，操作节奏变快

**下坡视觉效果**：
- 层级显示：向下箭头🔽（红色表示陡坡下坡，橙色表示缓坡下坡）
- 背景渐变：从上方向下方颜色变浅
- 海拔数值：显示海拔下降（不计入累积爬升）

**下坡地形障碍卡**：

| 卡牌名称 | 体能消耗 | 特殊效果 | 徒步数奖励 |
|---------|---------|---------|-----------|
| **缓坡下坡** | 3-5点 | 无特殊效果 | 50 |
| **陡坡下坡** | 2-4点 | 膝盖磨损风险（概率10%） | 100 |

**膝盖损伤机制**：
- 下坡时，每穿越3张陡坡下坡卡，检查一次膝盖磨损
- 膝盖磨损效果：疲劳+10，体能消耗+20%（持续5秒）
- 预防方法：使用登山杖（膝盖磨损概率-50%）
