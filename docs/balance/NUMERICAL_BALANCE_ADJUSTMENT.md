## 文档说明

本文档旨在为《大湾区徒步》游戏增加更真实的数值设定，基于实际徒步经验制定的基准标准，对现有属性系统、经济系统和关卡数值进行平衡调整，以追求更真实的数值感受和更自然的游戏体验。

**调整目标**：
1. 使游戏数值更贴近真实徒步体验
2. 保持游戏的可玩性和策略性
3. 确保数值系统的合理性和一致性
4. 提供清晰的数值转换规则

## 第一章：基准标准分析

### 1.1 基准标准定义

| 基准标准 | 数值含义 | 游戏映射 |
|---------|---------|---------|
| **初始体能 100 = 20 公里徒步** | 体能100点可支持20公里的徒步距离 | 体能消耗应与卡牌穿越数量对应 |
| **初始体能 100 = 2000 米海拔** | 体能100点可支持2000米的海拔爬升 | 海拔爬升应与层数对应 |
| **每 10 公里徒步 = 产生 20,000 徒步数** | 徒步数与徒步距离成正比 | 徒步数获得应与卡牌穿越数量对应 |
| **20 公里徒步 = 需要 2 次休息** | 徒步20公里需要休息2次 | 休息机制应与关卡层数对应 |
| **20 公里徒步 = 需要 3 瓶水 + 1 瓶运动饮料 + 1 次巧克力补给** | 补给需求与徒步距离对应 | 资源卡和商店物品应满足补给需求 |

### 1.2 基准换算关系

基于基准标准，我们得出以下换算关系：

**体能与距离的换算**：
```
1 点体能 = 20 公里 ÷ 100 = 0.2 公里
1 公里徒步 = 5 点体能
```

**体能与海拔的换算**：
```
1 点体能 = 2000 米 ÷ 100 = 20 米
20 米海拔爬升 = 1 点体能
```

**徒步数与距离的换算**：
```
1 公里徒步 = 20,000 徒步数 ÷ 10 公里 = 2,000 徒步数
1 点体能（0.2公里）= 400 徒步数
```

**休息与距离的换算**：
```
1 次休息 = 20 公里 ÷ 2 = 10 公里
1 次休息 = 50 点体能
```

**补给与距离的换算**：
```
20 公里徒步 = 3 瓶水 + 1 瓶运动饮料 + 1 次巧克力补给
每10公里 = 1.5 瓶水 + 0.5 瓶运动饮料 + 0.5 次巧克力补给
每5公里（25点体能）= 0.75 瓶水 + 0.25 瓶运动饮料 + 0.25 次巧克力补给
```

### 1.3 基准与现有系统的对比分析

**现有系统评估**：

| 对比项 | 现有设定 | 基准标准 | 差距评估 |
|-------|---------|---------|---------|
| **单次卡牌穿越消耗** | 地形障碍：3-30点体能 | 应消耗：根据地形类型计算 | 消耗范围合理，需细化 |
| **单局总距离** | 约50-120张卡牌 | 应支持20公里（初始体能100） | 卡牌数量需要大幅增加 |
| **单局海拔爬升** | 5-11层 × 30米 = 150-330米 | 应爬升2000米（初始体能100） | 海拔爬升过少 |
| **徒步数获得** | 无明确标准 | 每张卡牌400徒步数 | 需要建立明确标准 |
| **休息机制** | 无明确休息机制 | 每10公里（50点体能）需休息 | 需要新增休息机制 |
| **补给需求** | 无明确补给需求 | 20公里需要特定补给 | 需要建立补给机制 |

**关键问题识别**：

1. **卡牌数量不足**：现有关卡卡牌数量（50-120张）无法支持20公里的徒步体验
2. **海拔爬升过少**：现有关卡海拔（150-330米）远低于基准标准（2000米）
3. **徒步数获得不明确**：没有明确的徒步数获得标准
4. **缺少休息机制**：没有根据体能消耗触发休息的机制
5. **补给机制不完善**：没有根据距离和体能消耗建立补给需求

## 第二章：属性系统调整方案

### 2.1 体能系统调整

#### 2.1.1 体能消耗重新定义

**基础体能消耗模型**：

根据真实徒步体验，将卡牌穿越的体能消耗重新定义为：

| 卡牌类型 | 地形类型 | 体能消耗（点） | 对应距离（公里） | 对应海拔（米） |
|---------|---------|--------------|----------------|---------------|
| **风景卡** | 平坦道路 | 2 | 0.4 | 5 |
| **地形障碍** | 平坦道路 | 3 | 0.6 | 8 |
| **地形障碍** | 上坡路 | 8 | 1.6 | 20 |
| **地形障碍** | 陡坡 | 15 | 3.0 | 38 |
| **地形障碍** | 乱石路 | 12 | 2.4 | 30 |
| **地形障碍** | 溪流 | 8 | 1.6 | 20 |
| **地形障碍** | 悬崖栈道 | 20 | 4.0 | 50 |
| **资源卡** | 平坦道路 | 2 | 0.4 | 5 |
| **环境卡** | 平坦道路 | 2 | 0.4 | 5 |

**体能消耗计算公式**：
```
基础消耗 = 卡牌类型基础消耗 × (1 + 天气系数) × (1 + 负重系数) × (1 - 装备减免系数)

其中：
- 天气系数：晴天0，高温0.1，台风0.3，浓雾0.1
- 负重系数：负重/基础负重 × 0.1
- 装备减免系数：根据装备效果（如登山靴0.2，登山杖0.15）
```

#### 2.1.2 体能恢复机制调整

**自然恢复**：
```
恢复速率 = 基础速率 × 饥饿系数 × 口渴系数 × 疲劳系数 × 心率系数

其中：
- 基础速率：0.5 点/秒（降低原值，使恢复更慢）
- 饥饿系数：饥饿>80时1.0，50-80时0.5，0-49时0.05
- 口渴系数：口渴>80时1.0，50-80时0.5，0-49时0.02（更严格）
- 疲劳系数：疲劳<10时1.0，10-14时0.7，15-20时0.3
- 心率系数：心率<100时1.0，100-130时0.6，130-160时0.3，160-180时0.1
```

**休息恢复**：
```
休息恢复 = 25 点（休息一次恢复50点体能的50%）
休息时间：3秒（动画时间）
休息触发：体能降至50点时自动触发休息机会
休息次数限制：每关最多2次休息机会
```

#### 2.1.3 体能边界调整

| 边界类型 | 原设定 | 新设定 | 调整原因 |
|---------|-------|-------|---------|
| **最大值** | 100 | 120 | 允许超额恢复，提供缓冲 |
| **最小值** | 0 | 0 | 保持不变 |
| **警告阈值** | 30 | 50 | 提前警告，给玩家更多应对时间 |
| **危险阈值** | 10 | 20 | 提高危险阈值，增加紧张感 |

### 2.2 疲劳系统调整

#### 2.2.1 疲劳累积重新定义

**疲劳累积模型**：
```
疲劳累积 = 体能消耗 × 0.1 + 操作次数 × 0.1 + 心率影响 + 天气影响

其中：
- 体能消耗：当前行动消耗的体能
- 操作次数：连续穿越卡牌的数量（每5张卡牌操作次数+1）
- 心率影响：心率>130时，每超过10点疲劳+1
- 天气影响：高温天气疲劳+2，台风天气疲劳+3
```

**疲劳上限调整**：
```
原设定：0-20
新设定：0-30
调整原因：允许更长的连续行动时间，增加游戏策略性
```

#### 2.2.2 疲劳影响调整

| 疲劳范围 | 原影响 | 新影响 |
|---------|-------|-------|
| **0-9** | 无影响 | 无影响 |
| **10-14** | 无影响 | 体能消耗+10%，恢复速率-10% |
| **15-19** | 行动效率降低50% | 体能消耗+20%，恢复速率-30%，操作延迟+0.2秒 |
| **20-29** | 行动效率降低50% | 体能消耗+30%，恢复速率-50%，操作延迟+0.5秒 |
| **30** | 限制为20 | 强制休息，体能恢复速率-80% |

### 2.3 饥饿和口渴系统调整

#### 2.3.1 饥饿消耗重新定义

**饥饿消耗模型**：
```
饥饿消耗 = 基础消耗 × 体力消耗 × 0.02

其中：
- 基础消耗：0.1 点/秒
- 体力消耗：当前行动消耗的体能

示例：
- 穿越1张平坦道路卡牌（消耗3点体能）：饥饿消耗 3 × 0.02 = 0.06
- 穿越1张陡坡卡牌（消耗15点体能）：饥饿消耗 15 × 0.02 = 0.3
```

#### 2.3.2 口渴消耗重新定义

**口渴消耗模型**：
```
口渴消耗 = 基础消耗 × 体力消耗 × 0.03 × 天气系数

其中：
- 基础消耗：0.1 点/秒
- 体力消耗：当前行动消耗的体能
- 天气系数：晴天1.0，高温2.0，台风1.5

示例：
- 穿越1张平坦道路卡牌（消耗3点体能），晴天：口渴消耗 3 × 0.03 × 1.0 = 0.09
- 穿越1张陡坡卡牌（消耗15点体能），高温：口渴消耗 15 × 0.03 × 2.0 = 0.9
```

#### 2.3.3 恢复速率调整

**饥饿恢复速率**：
```
饥饿>80时：恢复速率100%
饥饿50-80时：恢复速率50%
饥饿0-49时：恢复速率5%
```

**口渴恢复速率**：
```
口渴>80时：恢复速率100%
口渴50-80时：恢复速率50%
口渴0-49时：恢复速率2%（更严格，强调饮水的重要性）
```

### 2.4 心率系统调整

#### 2.4.1 心率变化重新定义

**心率变化模型**：
```
心率变化 = 基础心率 + 体能消耗 × 2 + 疲劳 × 0.5 + 天气影响

其中：
- 基础心率：110（初始心率）
- 体能消耗：当前行动消耗的体能
- 疲劳：当前疲劳值
- 天气影响：高温+20，台风+15

示例：
- 穿越1张平坦道路卡牌（消耗3点体能），疲劳5，晴天：
  心率 = 110 + 3 × 2 + 5 × 0.5 + 0 = 118
- 穿越1张陡坡卡牌（消耗15点体能），疲劳15，高温：
  心率 = 110 + 15 × 2 + 15 × 0.5 + 20 = 162.5
```

**心率恢复**：
```
心率恢复 = (当前心率 - 基础心率) × 0.1 × 恢复系数

其中：
- 恢复系数 = 饥饿系数 × 口渴系数 × 疲劳系数
- 每秒恢复一次
```

#### 2.4.2 心率影响调整

| 心率范围 | 原影响 | 新影响 |
|---------|-------|-------|
| **80-100** | 无影响 | 无影响，恢复速率100% |
| **100-130** | 疲劳积累正常 | 疲劳积累正常，恢复速率80% |
| **130-160** | 疲劳积累+50% | 疲劳积累+100%，恢复速率50%，体能消耗+10% |
| **160-180** | 疲劳积累+100% | 疲劳积累+200%，恢复速率20%，体能消耗+20% |
| **180+** | 无（上限180） | 强制休息，心率上限调整为200，恢复速率10% |

### 2.5 负重系统重新设计

**负重系统**（新增）：

```
负重系统：影响体能消耗和疲劳累积

负重组成：
- 装备负重：每件装备有基础负重（如登山杖0.5kg，登山靴1.0kg，专业背包2.0kg）
- 物品负重：每件物品有负重（如水0.5kg/瓶，食物0.2kg/份）
- 资源负重：收集的资源有负重（如植物0.1kg/份）

负重上限 = 基础负重 + 体能 × 0.5

其中：
- 基础负重：15kg
- 体能100时负重上限：15 + 100 × 0.5 = 65kg

负重影响：
- 体能消耗增加：负重/负重上限 × 50%
- 疲劳累积增加：负重/负重上限 × 30%

示例：
- 当前负重30kg，负重上限65kg：
  体能消耗增加：30/65 × 50% = 23%
  疲劳累积增加：30/65 × 30% = 14%
```

### 3.4 补给需求机制（新增）

```
每10公里（50点体能）补给需求：
- 1.5 瓶水（恢复口渴30 × 1.5 = 45）

- 0.75 瓶水（恢复口渴22.5）
- 0.25 瓶运动饮料（恢复口渴7.5 + 恢复体能1.25）
```

**补给提示机制**：

## 第四章：关卡数值调整方案

### 4.1 关卡层数重新设计

#### 4.1.1 层数增加

**基于基准标准的层数计算**：

```
20公里徒步 = 100点体能
假设每层平均消耗10点体能（包含多种类型卡牌）
需要10层才能消耗100点体能

考虑以下因素：
- 玩家会使用补给和连击恢复部分体能
- 关卡难度递增，后期每层消耗更多
- 留有策略调整空间

调整后层数：
- 前段关（1-3）：15层
- 中段关（4-6）：20层
- 后段关（7-9）：25层
- BOSS关（10）：30层
```

#### 4.1.2 每层卡牌数量调整

**卡牌数量增加**：

| 关卡类型 | 总层数 | 每层卡牌数（从下到上） | 总卡牌数 | 对应距离（公里） |
|---------|-------|---------------------|---------|----------------|
| **前段关 (1-3)** | 15层 | 6-6-6-6-5-5-5-5-4-4-4-4-3-3-1 | 70 | 14公里（70×0.2） |
| **中段关 (4-6)** | 20层 | 7-7-7-7-6-6-6-6-5-5-5-5-4-4-4-4-3-3-2-1 | 95 | 19公里（95×0.2） |
| **后段关 (7-9)** | 25层 | 8-8-8-8-7-7-7-7-6-6-6-6-5-5-5-5-4-4-4-4-3-3-2-2-1 | 125 | 25公里（125×0.2） |

### 4.2 每层海拔重新定义

#### 4.2.1 海拔间距调整

**基于基准标准的海拔计算**：

```
2000米海拔 = 100点体能
假设平均每张卡牌消耗10点体能，每层10张卡牌
每层对应100点体能 = 2000米海拔

考虑以下因素：
- 不同地形类型海拔爬升不同
- 平坦道路海拔爬升较少，陡坡海拔爬升较多
- 海拔总爬升应与体能消耗对应

调整后海拔间距：
- 前段关（15层）：平均海拔爬升133米/层（2000÷15）
- 中段关（20层）：平均海拔爬升100米/层（2000÷20）
- 后段关（25层）：平均海拔爬升80米/层（2000÷25）
- BOSS关（30层）：平均海拔爬升67米/层（2000÷30）
```

**具体海拔设置**：

```
前段关（15层）：
- 第1层（山脚）：220米
- 第2层：220 + 133 = 353米
- 第3层：353 + 133 = 486米
- ...
- 第15层（山顶）：220 + 133 × 14 = 2082米

中段关（20层）：
- 第1层（山脚）：300米
- 第2层：300 + 100 = 400米
- 第3层：400 + 100 = 500米
- ...
- 第20层（山顶）：300 + 100 × 19 = 2200米

后段关（25层）：
- 第1层（山脚）：400米
- 第2层：400 + 80 = 480米
- 第3层：480 + 80 = 560米
- ...
- 第25层（山顶）：400 + 80 × 24 = 2320米

BOSS关（30层）：
- 第1层（山脚）：500米
- 第2层：500 + 67 = 567米
- 第3层：567 + 67 = 634米
- ...
- 第30层（山顶）：500 + 67 × 29 = 2443米
```

### 4.3 休息机制设计（新增）

#### 4.3.1 休息触发条件

**自动休息触发**：
```
条件1：体能降至50点时，显示休息提示，玩家可选择是否休息
条件2：连续穿越10张卡牌后，显示休息提示
条件3：疲劳达到20时，强制休息提示

休息限制：
- 每关最多2次休息机会
- 每次休息恢复25点体能
- 休息时间3秒（动画时间）
- 休息后重置疲劳为0
```

#### 4.3.2 休息效果

**休息效果**：
```
1. 体能恢复：+25点
2. 疲劳重置：归零
3. 心率恢复：降至基础心率
4. 饥饿/口渴：不受影响
5. 连击中断：休息后连击数清零
```

**休息动画**：
```
1. 显示"休息中..."提示
2. 角色坐下休息动画
3. 体能条动画：缓慢恢复25点
4. 疲劳条动画：快速归零
5. 心率动画：快速降至基础心率
6. 显示"休息完成！"提示
7. 恢复游戏
```

### 5.2 游戏体验预期变化

**更真实的徒步感受**：
1. 单局距离和海拔爬升大幅增加，更接近真实徒步体验
2. 补给需求明确，需要合理规划水和食物
3. 休息机制引入，需要合理分配体力
4. 负重系统引入，需要权衡装备和物品

**更深的策略性**：
1. 体能消耗更加精确，需要规划穿越顺序
2. 饥饿和口渴影响更大，需要及时补给
3. 心率系统更加严格，需要控制行动节奏
4. 疲劳累积更快，需要合理休息

**更高的挑战性**：
1. 关卡层数大幅增加，单局时间更长
2. 地形障碍率逐渐提高，难度递增更明显
3. 补给需求更高，资源管理更重要
4. 休息机会有限，需要谨慎使用

### 5.3 风险评估

**潜在风险**：
1. 单局时间过长，可能影响游戏节奏
2. 难度提升过快，可能导致新手玩家流失
3. 补给需求过高，可能导致资源稀缺
4. 休息机制引入，可能影响游戏流畅性

**风险缓解措施**：
1. 提供存档机制，允许玩家中途保存退出
2. 提供难度选择，新手模式降低卡牌数量和消耗
3. 调整资源卡掉落率，确保补给充足
4. 优化休息动画，减少等待时间
